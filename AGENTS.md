# AGENTS.md

This file contains guidelines and rules for AI agents working with this repository.

## Documentation Management

### Docs Folder Structure

All documentation files should be placed in the `docs/` subfolder.

### File Naming Convention

Files created in the `docs/` folder must follow this strict naming pattern:

```
{DATETIME}_{TYPE}_{name}.md
```

**Components:**

1. **DATETIME**: Use the format `YYYY-MM-DD-HHMM` (as generated by `date +"%Y-%m-%d-%H%M"`)
2. **TYPE**: Document type in ALL CAPS
3. **name**: Descriptive document name (can use hyphens for readability)

**Example:**
```
2025-11-18-1627_PLAN_new-unobtanium-detector.md
```

### Document Types

Common document types include (but are not limited to):

- `PLAN` - Planning documents
- `PROPOSAL` - Proposals for new features or changes
- `ANALYSIS` - Analysis documents
- `SUMMARY` - Summary documents
- `REFERENCE` - Reference materials
- `TASK` - Task descriptions and tracking
- `ISSUE` - Issue documentation and tracking

### Examples

```
2025-11-18-1627_PLAN_new-unobtanium-detector.md
2025-11-18-1630_PROPOSAL_refactor-auth-system.md
2025-11-18-1645_ANALYSIS_performance-bottlenecks.md
2025-11-18-1700_SUMMARY_sprint-retrospective.md
2025-11-18-1715_REFERENCE_api-endpoints.md
2025-11-18-1730_TASK_implement-feature-x.md
2025-11-18-1745_ISSUE_login-bug-investigation.md
```

### Rules

1. Always create documentation files in the `docs/` subfolder
2. Always use the prescribed naming convention
3. Use the current datetime when creating new documents
4. Choose the most appropriate TYPE for the document
5. Use descriptive, kebab-case names for the document name portion
